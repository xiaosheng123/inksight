# InkSight 硬件指南

本文档以**当前代码仓库**为准，说明推荐硬件方案、已内置的固件构建配置、接线方式和常见硬件问题。
如果你是第一次搭建，建议优先使用 **ESP32-C3 + 4.2寸墨水屏**。
这篇适合 **准备自己动手搭硬件、选板子、对接线和供电有疑问的人**。

## 1. 推荐方案（首选）

当前最推荐、也是默认固件配置的组合是：

- **主控**：ESP32-C3
- **屏幕**：4.2寸 SPI E-Paper
- **固件环境**：`epd_42_c3`

原因：

- 当前默认 `platformio` 环境就是 `epd_42_c3`
- 资料、文档、配置页预览都优先围绕 4.2寸版本展开
- 成本、体验、可复现性之间最均衡

## 2. 推荐 BOM

| 部件 | 推荐选型 | 说明 |
|------|----------|------|
| MCU | ESP32-C3 开发板 / SuperMini 类板型 | 低功耗、体积小、适合首发版本 |
| Display | 4.2寸 SPI 墨水屏 | 当前默认显示规格 |
| USB | USB 数据线 | 刷机和调试必须支持数据传输 |
| 连线 | 杜邦线 / 焊线 | 原型阶段可先用杜邦线 |
| 电源 | USB 直供（开发期） | 最稳定，排障方便 |
| 锂电池（可选） | `505060-2000mAh` | 长续航场景推荐选型 |
| 充电（可选） | TP5000 | 需配置为 LiFePO4 截止电压 |

典型 DIY BOM 仍然可以控制在 **约 220 元** 左右，但实际成本取决于屏幕采购渠道与外壳方案。

## 3. 当前代码内置的硬件构建配置

当前默认环境是 `epd_42_c3`，对外文档与产品流程也统一围绕 **4.2寸版本** 展开。

如果你需要查看代码中其他内置 profile，可以直接查看：

- `firmware/platformio.ini`

但如果你只是第一次搭建，建议直接使用默认的 **4.2寸方案**。

## 5. 4.2寸 + ESP32-C3 推荐接线

如果你使用推荐方案（`epd_42_c3`），建议按下表接线：

| ESP32-C3 | 墨水屏 |
|----------|--------|
| `GPIO4` | `CLK / SCK` |
| `GPIO6` | `DIN / MOSI` |
| `GPIO7` | `CS` |
| `GPIO1` | `DC` |
| `GPIO2` | `RST` |
| `GPIO10` | `BUSY` |
| `3V3` | `VCC` |
| `GND` | `GND` |

## 6. 供电建议

### 开发与排障

建议优先使用 **USB 直供**：

- 接线更简单
- 排障更容易
- 可以稳定使用串口日志

### 锂电池版本

如果你要做长期摆放版本，可以考虑：

- **单节 LiFePO4**
- **TP5000** 充电模块

当前推荐的锂电池型号是：

- **`505060-2000mAh`**

注意事项：

- TP5000 需要切换到 **LiFePO4 截止电压**配置
- 锂电池直驱 ESP32 前，请先确认整体电压范围与板卡稳定性
- 实际续航与刷新频率、Wi-Fi 质量、运行模式密切相关

目前项目对外通常描述为“低频刷新场景下可做到月级续航，理想情况下可到数月”，不要把所有使用场景都理解成固定 6 个月。

## 7. 组装建议

建议按下面顺序搭建：

1. 先用 **USB + ESP32 + 屏幕** 完成最小系统
2. 确认刷机成功、串口正常、屏幕能刷新
3. 再加入锂电池和充电模块
4. 最后考虑外壳、背板、夹层结构

如果你只是验证软件链路，可以完全跳过锂电池设计，先做 USB 供电版本。

## 8. 常见硬件问题

### 屏幕不显示

优先检查：

- `VCC / GND`
- `RST`
- `BUSY`
- SPI 接线是否与当前构建环境一致
- 屏幕是否为 SPI 版本，且与你选择的驱动/尺寸匹配

### 花屏、局部刷新异常

优先检查：

- `CLK / DIN / CS` 稳定性
- 线材是否过长
- 电源是否不稳

### 启动反复重启

通常优先怀疑：

- 供电不足
- 锂电池模块输出不稳定
- 刷机阶段请先切回 USB 供电排查

### Wi-Fi 正常但刷新慢

电子墨水屏刷新速度本身就比 LCD 慢，另外还会受以下因素影响：

- Wi-Fi 信号强度
- 当前模式是否需要外部数据或大模型
- 是否命中缓存

这通常不是“屏幕坏了”，而是系统整体链路时延的表现。

## 9. Busy Timeout 排查

如果串口中出现 `Busy Timeout!`，表示驱动在等待墨水屏 **BUSY** 引脚状态变化时超时。

优先排查：

1. **BUSY 没接或接触不良**
2. **BUSY 接错脚位**
3. **BUSY 被直接拉到 3.3V / GND**
4. **屏幕模组丝印与想象中的脚位顺序不一致**

对于 ESP32-C3 推荐方案，请重点确认：

- `GPIO10` ↔ 屏幕 `BUSY`

如果你使用的是其他板型，请以 `firmware/src/config.h` 中对应的 profile 为准。
